# Master Brief

Build a ListItUp login page that feels like the existing operational workspace rather than a marketing auth page.

The page must be a full-page 70/30 auth screen, not a centered mock card. The left 70% should be a clean React Bits LineWaves visual panel with the product copy anchored bottom-left. The right 30% should contain compact auth controls. It must use the full viewport width with premium spacing and breathable regions, while keeping the visual scale calm rather than zoomed-in. It should not have a page header, and the footer should stay minimal. It must keep one primary CTA, `Sign in`, with magic-link and reset-password paths subordinate. Sign-up should be available through an in-surface toggle that switches the form and adds Display Name without sending the User to a separate visual context. It should use the sample reference's dark compact shell, mono operational labels, orange action color, Lucide icons, and ambient canvas motion. The page should not include the removed right-side Inbox List or Verification Path preview.

Tension resolution: accessibility and performance stay hard floors; conversion clarity wins layout decisions; the signature moment is retained but kept secondary to the form.
