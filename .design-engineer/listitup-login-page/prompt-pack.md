# Prompt Pack

## Build Target

`mocks/login.html`

## Handoff Notes

- Use Tailwind CDN and simple HTML.
- Preserve ListItUp product language from `CONTEXT.md`.
- Treat the real app version as a full-page auth screen, not a centered mock shell.
- Use a 70/30 split on desktop: LineWaves visual panel with bottom-left copy on the left, compact auth controls on the right.
- Use the full viewport width with premium spacing and breathable regions.
- Do not include a page header; keep the footer minimal.
- Keep type and controls slightly zoomed out so the screen breathes.
- Use Lucide icons across the app UI.
- Primary CTA: `Sign in`.
- Include a Sign in / Sign up segmented toggle; sign-up mode adds Display Name and changes the CTA to `Create account`.
- Avoid a centered-card login pattern.
- Use one signature moment only: React Bits LineWaves on the left side behind the hero copy.
- Keep icons unboxed, placed directly on the dark background.
- Do not include the removed right-side Inbox List or Verification Path panels.
- Keep reduced-motion support for the canvas.
